Fran version 1.05

(Functional Reactive Animation)

Conal Elliott

Last edited October 06, 1997

I am tinkering with a new version of Fran. It's not really ready for general consumption yet, but in case you're really brave, you can try it out.

Notes on the new version:

  • As always, this works with Windows only.
  • You'll need DirectX version 3 or better. For NT, this means you need NT 4.0 Service Pack 3.
  • I changed over the representation of behaviors and events to use infinite trees and lists respectively (see src/BStack.hs). I'm writing a paper on the representation ideas. Unfortunately, this version is overly simplistic and ends up tree searching way too much. I'm working on a better version.
  • Unfortunately the performance is still pretty bad, and even noticeably worse than before. Fran should be running with GHC pretty soon, which should help performance a lot, though at the cost of less convenient development. Maybe the Fran source code could be improved to get acceptable performance under Hugs. Also, there are some mysterious space/time leaks. See the comment near the bottom of demos\Mover.hs.

Directions for trying out this new version:

  • Grab the zip file here. It's 622KB. Extract from the zip file, and you'll get a directory called "Fran1"..
  • Be sure you have a pretty recent version of Hugs. I'm using 970719.
  • Change your Hugs load path, via regedit or by using ":set -P" in Hugs, to include "..\\src", before any of the others. Otherwise, you'll probably end up loading the version of Fran that comes with Hugs.
  • Increase your heap size to 1Mb.
  • Copy HSpriteLib.dll to a directory on your search %PATH%.
  • Start it up. There are a few demos in Fran1\demos, notably Mover.hs. See "demo" in that source file. As I said above, Fran is rather slow right now.
